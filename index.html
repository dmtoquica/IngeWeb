Unidad 1: Fundamentos

1.1 ¿Qué es Ingeniería Web?

Disciplina centrada en el diseño, desarrollo y mantenimiento de aplicaciones de alta calidad mediante enfoques sistemáticos y cuantificables. No es solo "hacer páginas", es resolver problemas reales de forma eficiente.

1.2 Cómo funciona la Web

Basada en el modelo Cliente-Servidor. El cliente solicita vía HTTP/HTTPS y el servidor responde con contenido (HTML, CSS, JS). Incluye el uso de APIs y servicios REST.

1.3 Ciclo de Vida

  • Análisis de requisitos
  • Diseño conceptual (Wireframes)
  • Implementación (Código)
  • Pruebas y Despliegue
  • Mantenimiento (inherente al ciclo)

1.4 Roles del Proyecto

Equipos multidisciplinares: Ingenieros Web, Arquitectos, Diseñadores UX/UI, QA y DevOps colaboran para el éxito estratégico.

Unidad 2: Usuario y Experiencia (UX)

UX Básico

Cómo se siente el usuario. Se planifica desde el inicio mediante escenarios de uso y métodos de evaluación.

Usabilidad

Eficacia, eficiencia y satisfacción. Requiere navegación intuitiva e indicadores claros de uso.

Accesibilidad

Uso por personas con diferentes capacidades. Claves: atributos ARIA, navegación por teclado y contrastes.

Diseño Centrado en el Usuario (UCD)

Integramos al usuario desde el principio con prototipos y tests constantes para evitar errores costosos a futuro.

Navegación

Define el desplazamiento: mapas del sitio, menús jerárquicos y breadcrumbs (rutas semánticas).

Wireframes

Esquemas visuales sencillos para comunicar la estructura antes de escribir una sola línea de código.

Flujos de Usuario (User Flows)

Camino del usuario para cumplir objetivos: Registro → Login → Perfil → Acción.

Unidad 4: Implementación

HTML Semántico

Mejora accesibilidad y SEO. Usa etiquetas descriptivas:

<header> <nav>
<article> <section>
<footer>

CSS Moderno

Separa presentación de estructura. Conceptos clave:

  • Flexbox y Grid
  • Responsividad (RWD)
  • Variables y Preprocesadores

JavaScript

Añade interactividad dinámica que el diseño estático no puede lograr.

  • Manipulación del DOM
  • Gestión de Eventos
  • Peticiones Asíncronas (Fetch)

Buenas Prácticas

Código limpio, modularidad, uso de estándares W3C y control de versiones con Git.

Aviso de Implementación

La implementación no es solo código; es sostenibilidad.

Unidad 5: Calidad y Sostenibilidad

5.1 Pruebas

Verificar funcionalidad, rendimiento y resistencia a fallos inesperados.

Unitarias / Integración

5.2 Rendimiento

Optimización de recursos, minificación y caché para mejorar carga y SEO.

Web Vitals

5.3 Seguridad

Protección contra inyecciones, XSS, CSRF y manejo de autenticación segura.

Security by Design

5.4 Mantenimiento

Actualización constante, corrección de errores y adaptación a necesidades.

Evolución Continua
